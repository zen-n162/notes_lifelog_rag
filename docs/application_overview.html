notes_lifelog_rag

Apple Notes / iPhone Notes のエクスポートを、完全ローカルで検索・整理・振り返りできる 個人用ライフログ RAG アプリケーションです。単なる要約ではなく、出来事、考え、判断、 もう一度見返す価値のある記憶を、元メモの根拠と一緒に取り出します。

Local-first SQLite + FTS5 Embeddings + Reranker Evidence-aware AI outputs Gradio UI: 127.0.0.1:7860
目的 プライバシー設計 処理の流れ 主要機能 Gradio UI ローカルモデル CUDA実行 データとDB 差分実行 よく使うコマンド 現在の制限

目的

このアプリケーションは、ユーザーが Apple Notes からエクスポートしたメモを取り込み、 あとから「いつ何があったか」「その時何を考えていたか」「今なぜ見返す価値があるか」 を探しやすくするためのローカル専用メモ記憶アシスタントです。

AI が生成する出力は、必ず元メモへの evidence、推定の確からしさを示す confidence、見返す価値の強さを示す importance と一緒に扱います。 根拠が弱いものは断定せず、推測として読める形にします。

プライバシー設計

外部APIを使わない

OpenAI API、Gemini API、Claude API、クラウド embedding、クラウド vector DB などへ メモ本文や分析結果を送信しません。

Apple IDを扱わない

iCloud.com のスクレイピングや Apple ID 認証情報の保存は行いません。 入力はユーザーが明示的にエクスポートしたファイルです。

ローカルUI

UI のデフォルトホストは 127.0.0.1、ポートは 7860 です。 外部公開を前提にしていません。

本文表示は明示操作

一覧や検索結果では短い snippet を中心に表示し、元メモ本文はユーザーが詳細を開いた時に確認します。

処理の流れ

1. Import Phase 1-2

Markdown、TXT、HTML、JSON、PDF を読み、SQLite に保存します。

2. Search Phase 2-3

FTS5 / LIKE / embedding / reranker を組み合わせて候補を返します。

3. Analysis Phase 4

summary、category、event、thought を evidence 付きで構造化します。

4. Reflection Phase 5-6

月別 timeline と reflection を作り、UI で元メモまで確認できます。

5. Rediscovery Phase 7

QA Review、Analysis Health、Suggestions で見返す価値のある過去を再発見します。

主要機能

機能 説明 主な保存先
Ingestion エクスポート済み Apple Notes ファイルを取り込み、重複は content hash で避けます。 notes, note_chunks
Hybrid Search キーワード検索、semantic embedding、reranking を組み合わせます。 notes_fts, chunk_embeddings
Summaries タイトル、1行要約、詳細要約、重要ポイント、見返す理由を生成します。 note_summaries
Categories 研究、修論、就活、生活、感情・内省などのカテゴリを複数付与できます。 categories, note_categories
Events / Thoughts 出来事、予定、判断、気づき、価値観、不安、目標などを抽出します。 events, thoughts
Timeline / Reflections thoughts、events、summaries を優先して月別の出来事と考えを振り返り、coverage warningも表示します。 monthly_reflections
Suggestions / QA Review 重要な思考・出来事、見返す理由、低confidence、evidence不足をもとに再発見候補とレビュー候補を作ります。 suggestions, model_runs, import_errors

Gradio UI

UI は Apple HIG に近い落ち着いたダークモードの Notes-like デザインですが、 Apple Notes そのものではなく、Apple ロゴ、Apple Notes 商標、Apple 純正アイコンは使いません。 温かいチャコール背景、淡い黄色のアクセント、dark paper card、 3ペイン構成で、デフォルトでは 127.0.0.1:7860 で起動します。

Notes Workspace

左の記憶領域、中央のnote cards、右のpaper previewでメモを読み返します。

Sidebar

左ペイン上部で Library、Category、Month を選択し、その下で All Notes、Today Rediscovery、Important、Low Confidence、Evidence Missing、Analysis Healthを確認。

Note List

generated title、summary、category badges、confidence / importance、events/thoughts count、review badgeを表示。

Detail Pane

summary、important points、events、thoughts、evidence quotes、元メモ本文を紙面風に表示。

Analysis Jobs

embedding/analyze-all の dry-run と実行を明示ボタンで起動。heavy job は自動実行しません。

QA / Suggestions / Settings

低confidence、evidence不足、model failure、CUDA/model状態、Suggestions、Analysis Healthを確認。

一覧では個人メモ本文を大量表示せず、詳細ペインで選択したメモだけ原文を確認できます。 Search / Filter 後は中央のnote card自体をクリックすると選択でき、 選択カードのハイライトと右ペイン詳細が同期します。 Library と filter の選択状態は黄色の塗り潰しインジケータと淡い背景で分かるようにしています。 Phase 7 では QA Review、Timeline、Reflections、Suggestions の専用タブを追加し、 running analyze-all がある場合は UI の heavy job 実行を警告で止めます。 Workspaceで Suggestions を選んだ場合、中央に suggestion cards を表示し、 独立スクロールしながら候補を眺め、クリックした候補の根拠メモ本文を右ペインで確認できます。 QA Review、Timeline、Reflections、Suggestions の各タブでもカードを選択でき、 右ペインに元メモの summary、events、thoughts、evidence、Original Note を表示します。 Ask は検索 evidence に基づく控えめな回答を表示し、根拠ノートカードから詳細へ戻れます。

ローカルモデル

モデル設定は configs/models.yaml にあり、既存モデルパスがある場合だけ有効化します。 パスがない場合、または runtime dependency がない場合は disabled として扱い、関係ない機能は落としません。

  • Text generation: Qwen3-Swallow-8B, Qwen3-4B-Instruct
  • Embedding: Qwen3-Embedding-0.6B, ruri-v3-310m
  • Reranker: Qwen3-Reranker-0.6B, ruri-v3-reranker-310m
  • Tests: heavy model をロードしない mock backend を使用
モデルファイルは自動ダウンロードしません。hf download、 huggingface-cli download、モデル取得目的の wget / curl は、 ユーザーが明示した場合を除き実行しません。

CUDA実行とCPU fallback

build-embeddings、search、ask、analysis 系 CLI は --device auto を受け取り、CUDA が利用可能なら GPU、利用不可なら理由を表示して CPU fallback します。CUDA を必須にしたい場合は --require-cuda または --no-cpu-fallback を使います。

確認 / 実行 コマンド 目的
CUDA診断 python -m notes_lifelog_rag.cli cuda-status PyTorch CUDA build、driver、nvidia-smi、推定原因、推奨アクションを確認。
Embedding build-embeddings --backend local --only-missing --device auto --show-device GPUが使えればCUDA、使えなければCPUで差分embedding。
Analysis analyze-all --backend local --only-missing --device auto --dtype auto --max-new-tokens 512 model_runs cache と per-note commit を維持しながらローカルLLM実行。
NVIDIA driver が古い場合、アプリ側では根本解決できません。PyTorch CUDA build と driver の整合を取るか、 CPU fallback で続行してください。GPU OOM 時は --batch-size と --max-new-tokens を下げ、--limit で分割実行します。

データとDB

取り込んだメモ、chunk、embedding、分析出力はローカル SQLite DB に保存します。 生成データは data/processed/notes.db を中心に管理し、raw export や DB は Git 管理対象にしません。

領域 パス / テーブル 役割
Raw input data/raw/apple_notes_export/ ユーザーがエクスポートしたメモファイル置き場。
Database data/processed/notes.db notes, chunks, summaries, events, thoughts, model_runs などを保存。
Configs configs/app.yaml, configs/models.yaml アプリ設定、モデルパス、カテゴリ、プロンプト方針。

差分更新と再開安全性

877件規模のメモでも安全に少しずつ処理できるように、embedding build と analysis 系 CLI は --only-missing をデフォルトにしています。既存結果を確認して未処理分だけを選び、 progress bar を表示しながら batch / note 単位で commit します。 analysis 系 CLI の --limit は未指定なら全件候補を対象にし、少量テストしたい時だけ --limit 10 のように指定します。--dry-run は完全に読み取り専用で、 DB 行は追加・更新しません。

対象 skip 判定 再実行回避
Embeddings chunk_embeddings に同一 model / text hash の成功行があれば skip。 --dry-run で予定数確認、--force で意図的に再構築。
Summaries / Categories task / model / note id / content hash / prompt version が一致する成功 cache と保存済み行を skip。 model_runs の成功 JSON があれば LLM を再実行せず保存結果を復元。本文更新で content hash が変わると再分析対象。
Events / Thoughts 同じ hash の成功 cache と保存済み行を skip。保存行がなくても空抽出成功 cache は処理済み。 途中停止後も未保存 note だけを選び、成功済み cache を優先して使う。{"events":[]} / {"thoughts":[]} は空抽出として記録。

よく使うコマンド

conda activate notes_lifelog_rag
python -m notes_lifelog_rag.cli init-db
python -m notes_lifelog_rag.cli ingest-notes --input data/raw/apple_notes_export
python -m notes_lifelog_rag.cli cuda-status
python -m notes_lifelog_rag.cli build-embeddings --backend local --only-missing --device auto --batch-size 16 --show-device
python -m notes_lifelog_rag.cli build-embeddings --dry-run --limit 100
python -m notes_lifelog_rag.cli analyze-all --only-missing --dry-run
python -m notes_lifelog_rag.cli analyze-all --only-missing --limit 10
python -m notes_lifelog_rag.cli analyze-all --backend local --only-missing --device auto --dtype auto --batch-size 2 --max-new-tokens 512 --show-device
python -m notes_lifelog_rag.cli search "研究"
python -m notes_lifelog_rag.cli timeline --month 2026-05
python -m notes_lifelog_rag.cli reflections --month 2026-05
python -m notes_lifelog_rag.cli ui --host 127.0.0.1 --port 7860

現在の制限と今後の改善

  • UI の Ask は現時点では検索 evidence を使う軽量回答で、長文 LLM RAG 生成は今後の拡張対象です。
  • PDF は抽出可能なテキストのみ対象で、画像PDFのOCRはまだ通常メモ分析には組み込んでいません。
  • 月次 reflection は保存済み analysis output をもとに組み立てるため、先に analyze-all を実行すると内容が豊かになります。
  • 重いローカル LLM の実行は環境依存です。mock backend はテストと安全な開発用に残しています。
